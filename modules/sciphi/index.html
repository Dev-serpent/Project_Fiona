SciPhi — Scientific Operating System

SciPhi is a scientific investigation and simulation engine for Fiona. It is built around the Opsim (Optimized Simulation) Kernel — a reasoning engine that plans, constructs, executes, validates, and explains scientific investigations. It behaves like a lead scientist coordinating a research team, not like a function that computes equations.

SciPhi is not a general-purpose chatbot or a simple calculator. It is a structured scientific computation platform designed for reproducibility, transparency, and rigor.

Architecture

User Request → Opsim Kernel → Scientific Planner → Scientific Model
    → Problem Compiler → Solver Selection Engine → Solver
    → Validation → Uncertainty Analysis → Report Generation

The system is split into three independent layers:

Layer Responsibility Never knows about
Models Equations, variables, parameters, assumptions Solvers, computation
Solvers Numerical methods, convergence criteria Science, domains
Kernel Planning, compiling, selecting, validating Neither science nor numerics directly

Opsim Kernel (SciPhi/kernel/)

The central orchestrator that owns the full investigation lifecycle:

Module Component Responsibility
opsim.py OpsimKernel Orchestrator — runs the full pipeline
advisor.py SimulationAdvisor Checks if simulation is needed (or if analytical solution exists)
planner.py ScientificPlanner Determines domain, governing equations, mathematical form
compiler.py ProblemCompiler Translates scientific model → computational problem
solver_selector.py SolverSelectionEngine Matches problem form to solver capabilities
evaluator.py ScientificEvaluator Unit consistency, conservation laws, physical bounds
uncertainty.py UncertaintyAnalyzer Error propagation, confidence estimates
hypothesis.py HypothesisEngine Generates and evaluates testable hypotheses
provenance.py ProvenanceTracker Structured record of every scientific decision
report.py ReportGenerator Compiles all phases into structured investigation report

Scientific Models (SciPhi/models/)

Models describe only the science — equations, variables, parameters, assumptions, constraints, and domain-specific constants. They never reference solvers or numerical methods.

Domain Models Form
Physics Kinematics, Dynamics (harmonic oscillator), Thermodynamics (ideal gas), Electromagnetism (Coulomb/RC circuits), Quantum (particle in a box) ALGEBRAIC, ODE_INITIAL_VALUE, PDE, HYBRID
Chemistry Reaction kinetics (Arrhenius), Equilibrium (van't Hoff), Stoichiometry (mass/mole/yield) ODE_INITIAL_VALUE, ALGEBRAIC
Biology Logistic growth (population dynamics), SIR epidemic model ODE_INITIAL_VALUE
Earth Science Energy balance climate model (zero-dimensional) HYBRID
Engineering Structural analysis (beam bending/buckling), Circuit analysis (RC/RL) ALGEBRAIC, HYBRID

Total: 13 models across 5 domains.

All models implement the ScientificModel abstract base class from SciPhi.interfaces.model.

Solvers (SciPhi/solvers/)

Solvers describe only the computation — numerical methods, convergence criteria, and algorithmic parameters. They never reference scientific domains.

Solver Methods Forms
ODESolver RK4, Euler, DOPRI5 (adaptive) ODE initial value problems
AlgebraicSolver Newton-Raphson, Bisection, Fixed-point Algebraic systems
MonteCarloSolver Standard MC, Importance sampling Stochastic problems
SymbolicSolver SymPy solve/simplify/dsolve Symbolic mathematics
Optimizer Gradient descent, Nelder-Mead, BFGS Optimization problems

All solvers implement the Solver ABC from SciPhi.interfaces.solver and declare capabilities for automatic selection.

CLI Usage

# Full scientific investigation (Opsim pipeline)
fiona sciphi research "What is the trajectory of a projectile launched at 45 degrees?"

# Run a specific model directly with custom parameters
fiona sciphi simulate KinematicsModel --params '{"initial_angle": 45, "initial_velocity": 10}'

# Validate a simulation result against known data
fiona sciphi validate result.json

# List available models and solvers
fiona sciphi list-models
fiona sciphi list-models --domain physics
fiona sciphi list-solvers

Visualization

SciPhi includes an optional visualization module (SciPhi.visualization) for plotting simulation results:

  • plot_result(result) — time-series, bar, or heatmap plots from SimulationResult data
  • plot_report(report) — multi-panel figure with results, validation status, and uncertainty
  • plot_comparison(results, labels) — overlay multiple scenarios

All functions use dark theme styling and return base64 data URLs or save to file. Requires matplotlib (optional dependency).

Provenance

Every investigation produces a complete provenance record containing:

  • Models used and their versions
  • Governing equations and constants
  • Solver selection and configuration
  • Assumptions and approximations
  • Validation results
  • Uncertainty estimates
  • Full decision traceability
