fionaLocalPages — Web Dashboard

fionaLocalPages (fionaLocalPages/) is a browser-based operational dashboard for Fiona. It serves as a modern SPA control surface alongside the existing CLI and Tkinter GUI.

Architecture

Python backend (aiohttp) → HTML templates → JS modules (thin SPA layer)

Backend — server/app.py

Python aiohttp HTTP server with 24 handler modules and 115+ REST API endpoints at /api/v1/. Also serves WebSocket at /ws for real-time push and SSE at /api/v1/stream.

Frontend — Pages + Templates

27 page modules, each following the createPage(routeInfo) → { render, mount, destroy } pattern:

  • render() returns a mount-point <div> only
  • mount() is async — calls await loadTemplate('name', data) to fetch and inject HTML from templates/{name}.html, then binds events
  • destroy() cleans up intervals, listeners, and subscriptions

Template system (js/template-loader.js): - {{variable}} — escaped text interpolation - {{{rawHtml}}} — raw HTML (used for SVG icons) - {{#if}} / {{#each}} — conditionals and loops

The router also handles plain object exports ({ render, mount, destroy }) — this was a critical fix for blank pages when browser.js, terminal.js, and file-explorer.js exported objects instead of factory functions.

Terminal Implementation Detail

The terminal page (/terminal) runs all logic server-side:

  • _cwd is a module-level variable in handlers/terminal.py, initialized to os.path.expanduser("~")
  • Every safe_subprocess_run() call passes cwd=_cwd, so cd has actual persistent effect
  • Every response includes "cwd": _cwd — the JS client reads this to update its prompt display
  • GET /api/v1/terminal/cwd provides the initial directory on page mount
  • Autocomplete: POST /api/v1/terminal/autocomplete returns suggestions from a COMMAND_REFERENCE (~90+ commands, 14 categories)
  • JS is a thin input/output relay — no client-side cd tracking

Sidebar Implementation

The sidebar is static HTML (not a JS component) to avoid stale store subscriptions that caused the Sidebar.js component to accumulate duplicate listeners. Navigation uses data-action attributes dispatched through a global document event listener. The [data-action="toggle-sidebar"] handler is a single listener on the document, not on individual elements — this was the root cause fix for the sidebar toggle bug.

Running

./scripts/fiona-pages-start              # default port 8765
./scripts/fiona-pages-start --port 8080  # custom port
./scripts/fiona-pages-start --debug      # verbose logging

Open http://fiona.agent:8765 in a browser.

Status

All 27 pages fully implemented. Production concerns (authentication, HTTPS) still being addressed. This is a supplementary control surface — the CLI and Tkinter GUI remain the primary Fiona interfaces.
