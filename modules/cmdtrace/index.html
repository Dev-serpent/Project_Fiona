CmdTrace

CmdTrace provides a high-performance observability layer for Fiona. It records every action routed through the central ActionRouter into an append-only JSONL log and exposes analytics, search, export, compaction, and live-tailing.

Purpose

  • Audit Trail: Immutable record of every command executed via Fiona CLI, GUI, or remote ESP32 instructions.
  • Performance Monitoring: Records execution duration of every action for latency analysis.
  • Failure Analysis: Logs success/failure status and error messages for every routed task.
  • Operations: Statistics, advanced search, export to JSON/CSV, auto-compaction by size or age, and real-time tail monitoring.

Commands

Statistics

fiona action stats

Returns 14 fields: total/success/failure/unknown event counts, per-action breakdown, duration statistics (total/avg/max), time range, file size.

View recent history

fiona action history --limit 20

Filter history by action name

fiona action history --name host.status

Advanced search

fiona action search --query "timeout"           # Full-text search across all event fields
fiona action search --status ok                 # Only successful actions
fiona action search --status fail               # Only failed actions
fiona action search --regex "host\.\w+"         # Python regex match against serialized JSON
fiona action search --start-time 2026-01-01T00:00:00 --end-time 2026-06-30T23:59:59
fiona action search --limit 100

Export

fiona action export --format json               # Pretty-printed JSON array
fiona action export --format csv                # CSV with auto-discovered columns
fiona action export --output /path/to/export.json

Compact / Rotate

fiona action compact                            # Default: 10 MB max, 30 days, keep last 10k
fiona action compact --max-size-mb 5 --max-age-days 7 --keep-last 1000

Returns a report: original/new size, original/new entry count, how many trimmed by size vs age.

Clear the trace log

fiona action clear

Live tail (follow new entries)

fiona action tail                               # Follow new entries in real-time
fiona action tail --interval 0.5                # Poll every 0.5 seconds

API Endpoints

Method Endpoint Description
GET /api/v1/actions/history Recent trace events (query: limit, name)
GET /api/v1/actions/stats Trace analytics (14-field statistics object)

TUI Integration

The History page in the fiona cli TUI provides a real-time sliding view of the latest system activity using CmdTrace.read_trace().

Storage

Data is stored as JSON Lines (one JSON object per line) in:

~/.config/fiona/cmdtrace.jsonl

Auto-compaction

When trace_compact() is called (or configured to run periodically), the file is checked against size and age thresholds. Entries beyond the configured limits are trimmed, with the most recent keep_last entries preserved.

Python API

from CmdTrace import (
    append_trace,      # Append an event to the trace
    read_trace,        # Read recent events (filter by action_name, limit)
    clear_trace,       # Delete the trace file
    trace_stats,       # Get 14-field analytics dictionary
    trace_search,      # Advanced search (query, status, regex, time range, limit)
    trace_export,      # Export trace to JSON or CSV file
    trace_compact,     # Compact/rotate by file size or entry age
    trace_tail,        # Iterator that yields new events as they're written
)

trace_stats(path) -> dict

Returns: - total_events, skipped_lines — event counts - per_action — {action_name: count} sorted by frequency - success_count, failure_count, unknown_status_count - first_event_time, last_event_time, time_span_seconds - total_duration, avg_duration, max_duration - file_size_bytes, file_size_human

trace_search(*, query, action_name, status, start_time, end_time, regex, limit, path) -> list[dict]

All parameters are keyword-only: - query — case-insensitive substring search across ALL string values - action_name — exact match on event["action"] - status — True for ok, False for failures, None for no filter - start_time / end_time — ISO timestamp range filter - regex — Python re.search against json.dumps(event) - limit — max results (default 100)

trace_export(path, output, fmt) -> Path

  • fmt="json" — write as pretty-printed JSON array
  • fmt="csv" — write as CSV with auto-discovered headers
  • If output is None, generates cmdtrace-export-{timestamp}.{fmt}

trace_compact(*, max_size_mb, max_age_days, keep_last, path) -> dict

Returns compaction report with original_size_bytes, new_size_bytes, original_entries, new_entries, trimmed_by_size, trimmed_by_age.

trace_tail(path, interval) -> Iterator[dict]

Returns an iterator that polls for file growth every interval seconds. Handles file truncation gracefully. Call .close() to stop.
