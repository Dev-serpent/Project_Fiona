RecallVault

RecallVault is Fiona's persistent "remembrance" store — a key-value database with categories, tags, TTL-based expiry, import/export, timestamped backups, and optional TF-IDF semantic search.

Purpose

  • Long-term Memory: Store facts, configuration snippets, or notes that persist across sessions.
  • Categorization & Tagging: Group information into categories and attach multiple tags for rich filtering.
  • TTL-based Expiry: Entries can auto-expire after a configurable number of seconds.
  • Portability: Import and export entries as JSON or CSV files.
  • Safety: Automatic timestamped backups before any destructive write.
  • Semantic Search: Pure-Python TF-IDF index ranks results by relevance (no external dependencies).

Commands

Save a remembrance

fiona recall remember "phi-host" "192.168.1.10" --category network
fiona recall remember "db-password" "s3cret" --tags secret,prod --ttl-seconds 86400
fiona recall remember "theme" "dark" --category gui --tags ui,preferences

Options: --category (default: "general"), --tags (comma-separated list), --ttl-seconds (auto-expire after N seconds).

Search remembrances

fiona recall search "phi"                      # Searches key, value, category, AND tags
fiona recall search                             # List all entries

List categories

fiona recall categories

Statistics

fiona recall stats

Returns 12 fields: total entries, per-category and per-tag counts, storage size (bytes + human-readable), oldest/newest entry, expired/tagged/untagged counts, categories list.

Export

fiona recall export --format json               # Pretty-printed JSON array
fiona recall export --format csv                # CSV with headers: key,value,category,tags,created_at,ttl_seconds
fiona recall export --output /path/to/export.json

Import

fiona recall import /path/to/entries.json       # Auto-detects format from extension
fiona recall import /path/to/entries.csv

JSON format: list of objects with key, value, optional category, tags, ttl_seconds. CSV format: headers key,value,category,tags,created_at,ttl_seconds. Imported entries with existing keys overwrite the stored value.

Backup

fiona recall backup                             # Creates recallvault-{timestamp}.json

Forget an entry

fiona recall forget "phi-host"

Clear the vault

fiona recall clear

API Endpoints

Method Endpoint Description
GET /api/v1/recall/search?q=... Search entries (optional query)
POST /api/v1/recall/remember Save/replace entry (key, value, category?, ttl_seconds?, tags?)
DELETE /api/v1/recall/forget/{key} Remove entry by key
GET /api/v1/recall/stats Vault statistics (12-field object)

TUI Integration

The Recall page in the fiona cli TUI displays the most recent snippets from the vault and updates in real-time.

Storage

Data is stored as a JSON array of objects in:

~/.config/fiona/recallvault.json

Backups are written to ~/.config/fiona/recallvault-{YYYYMMDDTHHMMSS}.json.

Python API

from RecallVault import (
    DEFAULT_RECALL_PATH,
    RecallEntry,
    remember,          # Save/replace an entry (key, value, category?, ttl_seconds?, tags?)
    search_recall,     # Search by key/value/category/tags
    forget,            # Remove entry by key
    list_categories,   # List unique categories
    clear_recall,      # Delete the vault file
    load_recall,       # Load all entries (auto-purges expired)
    recall_stats,      # Get 12-field analytics dictionary
    export_recall,     # Export vault to JSON or CSV file
    import_recall,     # Import entries from JSON or CSV file (merges)
    backup_recall,     # Create timestamped backup copy
    SemanticIndex,     # TF-IDF semantic search index
)

RecallEntry dataclass fields

Field Type Default Description
key str (required) Unique entry identifier
value str (required) Stored value
category str "general" Single category label
tags list[str] \| None None Optional list of tags
ttl_seconds int \| None None Time-to-live in seconds (None = no expiry)
created_at str (auto) ISO-8601 creation timestamp
accessed_at str "" Last access timestamp (updated on search)

remember(key, value, *, category, ttl_seconds, tags, path) -> Path

Saves or replaces an entry. If ttl_seconds is set, the entry will be auto-purged on the next load_recall() call after that many seconds have elapsed since created_at.

load_recall(path) -> list[RecallEntry]

Loads all entries, filtering out any that have exceeded their TTL. Expired entries are silently purged and the file is rewritten.

recall_stats(path) -> dict

Returns: - total_entries, per_category, per_tag - total_storage_bytes, total_storage_human - oldest_entry (key + created_at), newest_entry - expired_entries (count that would be purged by TTL) - tagged_entries, untagged_entries - categories

export_recall(path, output, fmt) -> Path

  • fmt="json" — write as pretty-printed JSON array
  • fmt="csv" — write as CSV with headers
  • If output is None, generates recallvault-export-{timestamp}.{fmt}

import_recall(path, input, fmt) -> int

  • Auto-detects format from file extension if fmt is None
  • Merges with existing entries (same key = overwrite)
  • Returns count of imported entries

backup_recall(path) -> Path

Copies the vault file to recallvault-{YYYYMMDDTHHMMSS}.json in the same directory. Creates an empty backup if the vault doesn't exist.

SemanticIndex (TF-IDF search)

Pure-Python TF-IDF implementation with zero external dependencies:

from RecallVault import SemanticIndex

index = SemanticIndex()
index.build(entries)                       # Build TF-IDF matrix from entry dicts
results = index.search("query", limit=5)   # Returns [{key, score}, ...] sorted by relevance
index.add_entry("key", "text")             # Marks index for rebuild
  • Tokenizes by word boundaries, lowercases
  • Computes term frequency / inverse document frequency
  • Scores by cosine similarity of TF-IDF vectors
  • Gracefully degrades (empty results) if no documents indexed
